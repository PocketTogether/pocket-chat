# 前端全局关闭浏览器拼写检查（spellcheck）

**适用场景：** 输入框出现红色波浪线、浏览器自动提示拼写错误、组件库输入框默认开启拼写检查等。

---

## 1. 浏览器的拼写检查是什么

现代浏览器会对以下元素自动执行拼写检查：

- `<input>`
- `<textarea>`
- `contenteditable` 区域

表现为：

- 红色波浪线
- 自动拼写纠正提示
- 自动替换（部分浏览器）

这些行为是浏览器层级的，不受 CSS 控制。

---

## 2. 关闭拼写检查的标准方式：`spellcheck` 属性

HTML 标准提供了一个全局属性：

```html
spellcheck="false"
```

它可以用于任何可编辑元素。

示例：

```html
<input spellcheck="false">
<textarea spellcheck="false"></textarea>
<div contenteditable spellcheck="false"></div>
```

---

## 3. 全局关闭拼写检查的最佳方式：在 `<html>` 上设置

浏览器允许从根节点继承 `spellcheck` 状态，因此可以直接在 `<html>` 上关闭：

```html
<html spellcheck="false">
```

效果：

- 所有输入框、textarea、contenteditable 默认关闭拼写检查
- 除非某个子元素显式写了 `spellcheck="true"`
- 兼容 Chrome / Edge / Firefox / Safari

这是最接近“全局关闭”的方式。

---

## 4. 为什么 CSS 无法关闭 spellcheck

CSS 无法控制浏览器的拼写检查行为，例如：

```css
input {
  spellcheck: false; /* 无效 */
}
```

原因：

- 拼写检查属于浏览器行为，不属于渲染层
- CSS 只能控制视觉样式，不能控制浏览器内部逻辑

---

## 5. 如果页面动态生成输入框：用 JS 自动关闭

如果项目中大量使用组件库（如 Element Plus、AntD、Material UI），可以用 JS 自动处理：

```js
document.addEventListener("DOMContentLoaded", () => {
  document.querySelectorAll("input, textarea, [contenteditable]")
    .forEach(el => el.setAttribute("spellcheck", "false"));
});
```

若页面会动态插入输入框，可使用 MutationObserver：

```js
const observer = new MutationObserver(() => {
  document.querySelectorAll("input:not([spellcheck]), textarea:not([spellcheck]), [contenteditable]:not([spellcheck])")
    .forEach(el => el.setAttribute("spellcheck", "false"));
});

observer.observe(document.body, { childList: true, subtree: true });
```

---

## 6. Vue / React 项目中的全局关闭方式

### Vue

在 `App.vue` 或全局布局组件：

```html
<template>
  <div spellcheck="false">
    <router-view />
  </div>
</template>
```

或封装全局 Input 组件：

```vue
<input v-bind="$attrs" spellcheck="false">
```

### React

在根组件：

```jsx
<html spellCheck={false}>
```

或封装 Input：

```jsx
const Input = props => <input spellCheck={false} {...props} />
```

---

## 7. 最终建议

优先级从高到低：

1. **在 `<html>` 上设置 `spellcheck="false"`（最推荐）**
2. 封装全局 Input 组件
3. 用 JS 自动处理动态生成的输入框
4. 单独在每个输入框上设置（最麻烦）
