# 📘 关于 `100vh` 与 `100dvh` 的前端布局笔记

（含 Tailwind：`h-screen` vs `h-dvh`）

## 🎯 为什么要讨论 `100vh`？

在桌面端，`100vh` 表示视口高度的 100%，没有任何问题。  
但在移动端浏览器中，视口高度会随着**地址栏、工具栏的出现与收起**而动态变化。

这导致一个经典问题：

> 使用 `100vh` 时，页面底部内容（如底栏、按钮、提交区）可能被浏览器 UI 遮挡，用户必须滚动一下才能看到。

这是移动端最常见、最恼人的布局坑之一。

---

# 📏 机制：为什么 `100vh` 会出问题？

移动端浏览器存在三种“视口高度”概念：

|单位|含义|
|---|---|
|`vh`|旧视口高度，包含浏览器 UI，不等于真实可视区域|
|`svh`|Small Viewport Height（UI 完全展开时）|
|`lvh`|Large Viewport Height（UI 完全收起时）|
|`dvh`|Dynamic Viewport Height（**实时可视区域高度**）|

传统的 `100vh` 实际上更接近 **lvh**，而不是用户当前真正看到的区域。

因此：

- 地址栏展开 → 可视区域变小，但 `100vh` 不变
- 结果：底部内容被挡住

---

# ✅ `100dvh`：移动端的正确选择

`100dvh` 表示：

> **当前时刻用户真实可视区域的高度**

它会随着浏览器 UI 的变化实时更新：

- 地址栏展开 → `100dvh` 变小
- 地址栏收起 → `100dvh` 变大
- 页面高度始终与真实可视区域一致
- 不会出现底部被遮挡的问题

这正是移动端布局真正需要的行为。

---

# 🧪 实际体验对比

### 使用 `100vh`

- 初次进入页面：底部按钮被遮挡
- 滚动后地址栏收起：页面突然变长，布局跳动
- 体验割裂、不稳定

### 使用 `100dvh`

- 初次进入：底部内容完全可见
- 滚动时：高度自然变化，无跳动
- 布局稳定、符合用户预期

---

# 🛠️ Tailwind 对应写法

Tailwind 已内置：

```html
<div class="h-screen"> <!-- 等于 100vh -->
<div class="h-dvh">   <!-- 等于 100dvh -->
```

移动端布局中，**应该优先使用 `h-dvh`**。

---

# 🖥️ 那 PC 端呢？`100dvh` 会不会有问题？

不会。

在 PC 浏览器中：

- 地址栏高度固定
- 可视区域不会动态变化
- `100vh` 与 `100dvh` 数值一致

因此：

> **PC 端使用 `100dvh` 完全没问题，甚至更推荐，因为语义更明确、跨设备更一致。**

你可以放心在所有设备上统一使用 `100dvh`。

---

# 📌 最佳实践（可直接放进团队规范）

## 📐 全屏布局统一使用 `100dvh`（或 Tailwind 的 `h-dvh`）

理由：

- 避免移动端底部内容被遮挡
- 避免地址栏收起导致的布局跳动
- 保持 PC / Mobile 一致的心智模型
- 更符合 CSS Viewport Units Level 4 标准
- 更语义化、更未来向

示例：

```css
height: 100dvh;
```

Tailwind：

```html
<div class="h-dvh">
```

---

# 🧩 兼容旧浏览器（可选）

```css
height: 100vh;
height: 100dvh;
```

新浏览器使用 `100dvh`，旧浏览器 fallback 到 `100vh`。
