# ✅ 第一部分：排序逻辑是否正确？

你的排序代码：

```ts
result.sort((a, b) => {

    psEarliest.get(a.id)?.created ??
    earliestMap.get(a.id)?.created ??
    '9999-12-31'

    earliestMap.get(b.id)?.created ??
    '9999-12-31'

```

### ✔ **排序逻辑完全正确**

原因：

- 你要按“最早出现的 presence 记录”排序
- earliestMap 和 psEarliest 都是 Map<userId, earliestRecord>
- created 是 ISO 字符串
- ISO 字符串按字典序排序 = 按时间排序
- fallback `'9999-12-31'` 会把没有 earliest 的用户排到最后
- localeCompare 对 ISO 字符串排序是正确且稳定的

### ✔ **不会出现排序错误**

因为 ISO 时间格式保证了字典序与时间序一致。

### ✔ **不会出现 undefined 排序问题**

你已经用 `?? '9999-12-31'` 兜底。

# ⚠️ 第二部分：排序逻辑是否有潜在问题？

只有一个小点值得注意：

### 🔸 你每次排序时都在 Map 中查找 created

这意味着：

- 每次排序比较时会访问 Map 两次
- 排序复杂度是 O(n log n)
- 每次比较又访问 Map → 总体 O(n log n) * O(1) = O(n log n)

这在用户数量不大的情况下（几十、几百）完全没问题。

如果用户数量达到 **几千以上**，你可能希望提前把 earliest 时间缓存成一个 Map：

```ts
const earliestCreatedMap = computed(() => {
  const map = new Map<string, string>()
  for (const [id, rec] of psEarliest) map.set(id, rec.created)
  for (const [id, rec] of earliestMap) if (!map.has(id)) map.set(id, rec.created)
  return map
})
```

然后排序时直接：

```ts
const ea = earliestCreatedMap.value.get(a.id) ?? '9999-12-31'
```

但这只是优化，不是必须。
