#### useRealtimeUsersStatusComputed（AITODO 完整版）

```
文件 src/composables/realtime-users-status/computed.ts

const psStore = useRealtimeUsersPresencesStatusStore()
const nvmStore = useRealtimeUsersNotViewingMarksStore()
const up3815Query = useUsersPresencesStatusInitGetListQuery()

nowRefTimestampNumber: ref<number>
  外部会定时更新（例如每秒更新一次）

getStatusByUserId(userId)
  slm4708 = psStore.statusLatestMapByUser.get(userId)
  mm4731 = nvmStore.marksMapByUser.get(userId)
  uq4812 = up3815Query.data?.items.find(x => x.author === userId)

  若三者都不存在 → return null

  取三者中 created 最大的作为最终状态来源
  若 mm4731.created > slm4708.created → isNotViewing = true
  否则若 slm4708 存在 → isNotViewing = slm4708.isNotViewing
  否则若 uq4812 存在 → isNotViewing = uq4812.isNotViewing || false

  isTyping:
    若 slm4708 存在 → slm4708.isTyping
    否则 false

  statusIsoDate:
    slm4708?.created || uq4812?.created || mm4731?.created

  return {
    isNotViewing
    isTyping
    statusIsoDate
  }

getStatusByUserIdWithIsOnline(userId)
  const s = getStatusByUserId(userId)
  若 s 为 null → return null

  const diff = nowRefTimestampNumber.value - new Date(s.statusIsoDate).getTime()
  const isOnline = diff <= 100000  // 100 秒

  return {
    ...s,
    isOnline
  }
```

#### AITODO：全部在线用户（已完成）

```
allOnlineUsers: computed(() => {
  const arr = []

  遍历 psStore.statusLatestMapByUser.keys() 与 nvmStore.marksMapByUser.keys()
  合并成一个 userIdSet

  对每个 userId:
    const s = getStatusByUserIdWithIsOnline(userId)
    若 s?.isOnline === true，则 push({ userId, ...s })

  排序规则：
    1. 优先按 statusEarliestMapByUser 中的 created 升序（越早出现越靠前）
    2. 若 earliest 不存在，则按 statusIsoDate 升序
    3. 若都不存在（极少情况），按 userId 字典序

  return arr
})
```

排序逻辑解释：  
“越早有状态记录的用户越靠前” → earliestMap 是最可靠的依据。  
若 earliestMap 没有（例如该用户只在 initQuery 中出现），则 fallback 到 statusIsoDate。

#### AITODO：最近活跃用户（已完成）

```
recentActiveUsers: computed(() => {
  const arr = []
  const userIdSet = new Set()

  将 psStore.statusLatestMapByUser.keys() 加入 userIdSet
  将 nvmStore.marksMapByUser.keys() 加入 userIdSet
  将 up3815Query.data?.items.author 加入 userIdSet

  对每个 userId:
    const s = getStatusByUserIdWithIsOnline(userId)
    若 s?.isOnline === false，则 push({ userId, ...s })

  排序规则：
    1. 优先按 statusEarliestMapByUser 中的 created 升序
    2. 若 earliest 不存在，则按 statusIsoDate 降序（最近活跃的排前面）
    3. 若都不存在，则按 userId 字典序

  return arr
})
```

排序逻辑解释：  
最近活跃用户 = 不在线但有记录的用户。  
排序优先级：

- earliestMap 用于“用户出现顺序”
- 若 earliest 不存在，则按最近活跃时间排序（最近的排前面）
