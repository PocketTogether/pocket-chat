## 全部用户的实时状态订阅接收与处理 这一块

### 命名确定与简单描述
```
用于统一处理全部用户的实时状态，包括订阅 PocketBase 的两个集合、
将事件写入本地 store、并提供对外可用的实时状态数据。
```

有两个pinia store，注意是setup组合式的
```
一个是 useRealtimeUsersPresencesStatusStore
文件 src\stores\realtime-users-presences-status.ts
useRealtimeUsersPresencesStatusStore
用于储存 userPresencesStatus 开始订阅以来所得到的，最旧记录 与 最新记录（在线、是否可见、是否输入中）。

一个是 useRealtimeUsersNotViewingMarksStore
文件 src\stores\realtime-users-not-viewing-marks.ts
useRealtimeUsersNotViewingMarksStore
用于储存 userNotViewingMarks 的最新记录（基于 sendBeacon 的不可见标记）。
```

有两个组合式函数，将会使用上面这个个store
```
useRealtimeUsersStatusSubscription
文件 src\composables\realtime-users-status\subscription.ts
用于启动订阅的组合式函数
其中有一个启动订阅函数
其中调用pb，收到事件时判断并调用store提供的方法来储存数据
还会配合两个store中专门用于标记订阅是否已启动的响应式布尔值来避免重复订阅

useRealtimeUsersStatusComputed
文件 src\composables\realtime-users-status\computed.ts
用于供其他地方使用实时数据的组合式函数
就是会使用两个store中的数据，有一些函数和计算属性以供外界使用
```

## 用户自己的实时状态循环检查与发送 这一块

### 命名确定与简单描述
```
用于管理用户自己的实时状态，包括本地状态管理（在 store 中）、
间隔 5 秒的无限循环判断与发送逻辑，以及 visibilitychange(hidden) 时的 sendBeacon 发送。
```

有一个pinia store，注意是setup组合式的
```
useSelfPresenceStore
文件 src/stores/self-presence.ts
用于储存本地状态：是否输入中、上次发送内容、上次发送完成时间戳、当前是否正在发送、是否已启动循环等。
```

三个组合式函数，会使用上面这个store
```
useSelfPresenceDispatcher
文件 src/composables/self-presence/dispatcher.ts
用于封装判断与发送逻辑，提供方法供外界调用，
如：判断是否需要发送、执行发送、立即触发一次判断与发送等。

useSelfPresenceLoopAndViewingHook
文件 src/composables/self-presence/loop-hook.ts
用于在 app.vue 中，调用本组合式函数中的某个函数，启动无限循环与监听 visibilitychange，
内部会配合 store 中的响应式布尔值标记避免重复启动，
会使用 useSelfPresenceDispatcher ，循环中会调用SelfPresenceDispatcher中的方法。

useSelfPresenceTypingReporter
文件 src/composables/self-presence/typing-reporter.ts
用于供输入栏组件调用，更新 store 中的输入状态，
还能在在第一次输入或发送后调用SelfPresenceDispatcher中的方法，以便立即判断并决定是否立即发送。
```

#### useSelfPresenceDispatcher
```
useSelfPresenceDispatcher
文件 src/composables/self-presence/dispatcher.ts

将使用 useSelfPresenceStore
如 const selfPresenceStore = useSelfPresenceStore()

isTypingFn 函数
根据 nowRefTimestamp 与 isTypingMark 与 isTypingMarkTimestamp 判断
isTypingMark 为 false 则为 false
isTypingMark 为 true 且 isTypingMarkTimestamp 距今不超过5秒才为 true

isNotViewingFn 函数
判断 document.visibilityState !== 'visible'

checkNeedSendPresenceByUpdatePresence 函数，是否需要发送以更新实时状态
根据 isTyping isNotViewing 与 lastPresenceSentPayload 判断是否需要发送
即状态改变时需要发送

checkNeedSendPresenceByKeepOnline 函数，是否需要发送以维持在线
根据当前时间戳与 lastPresenceSentTimestamp 判断是否需要发送，超过50秒需发送

checkNeedSendPresenceByIsPresenceSentSuccess 函数
根据上次发送是否成功，判断是否需要发送。上次发送如果不成功则需要发送

checkCanSendPresenceByLastPresenceSentTimestamp 函数
根据上次发送时间，来判断是否能发送。上次发送如果距今少于两秒则不能发送

checkCanSendPresenceByIsPresenceSending 函数
根据是否正在发送，来判断是否能发送。正在发送则不能发送

sendPresenceFn 函数
只供 sendPresenceIfNeedAndCanSend 使用，不提供给外部使用
将调用api函数 pbUsersPresencesStatusCreateApi
将根据设计对store进行操作

sendPresenceIfNeedAndCanSend 函数，核心发送函数，供循环与其他地方调用
if (
  (
    checkCanSendPresenceByIsPresenceSending() &&
    checkCanSendPresenceByLastPresenceSentTimestamp()
  ) &&
  (
    checkNeedSendPresenceByKeepOnline() ||
    checkNeedSendPresenceByUpdatePresence() ||
    checkNeedSendPresenceByIsPresenceSentSuccess()
  )
) {
  满足才真正进行发送 sendPresenceFn
}


checkCanSendNotViewingMarksByIsNotViewing 函数
检查 isNotViewingFn ，页面不可见才能发送

checkNeedSendNotViewingMarksByPresence 函数
（lastNotViewingMarksSentTimestamp 小于 lastPresenceSentTimestamp）且
（lastPresenceSentPayload 的 isNotViewing 为 false）才能发送
即已经isNotViewing=ture时就不必发送了

sendNotViewingMarksFn
只供 sendNotViewingMarksIfNeedAndCanSend 使用，不提供给外部使用
将调用api函数 pbUsersNotViewingMarksCreateApiBySendBeacon
将根据设计对store进行操作

sendNotViewingMarksIfNeedAndCanSend 函数供visibilitychange钩子中调用，
实际绑定是在useSelfPresenceLoop中的启动函数中，即差不多和Presence循环检查是在一起的
if (
  checkCanSendNotViewingMarksByVisibilityState() &&
  checkNeedSendNotViewingMarksByPresence()
) {
  满足才真正进行发送 sendNotViewingMarksFn
}

return {
sendPresenceIfNeedAndCanSend
sendNotViewingMarksIfNeedAndCanSend
}

```
