## 详情说明

### 成员
- 成员页
- 成员详情页

```
成员页（成员列表页）、显示全部成员、可控制升序或降序排序
页面样式类似文件选择页，即双列布局，还有一点区别是不需要底栏

成员详情页，类似文件详情页，不过样式布局应改改
显示用户名、用户简介
显示用户当前正在播放的
显示用户发送的消息
显示用户创建的播放列表
显示用户播放记录

```

### 动态
- 动态页

```
显示在线成员，并显示状态
用户头像右下角，有表示用户当前状态的圆点图标，主要给聊天列中的用户头像和动态页的在线用户添加就好

显示播放列表
- 全局所有正在播放的播放列表-主持播放者，点击即可跟随播放
- 热门的几个播放列表
- 最新的几个播放列表
- 附带导航，可跳转至播放列表页

```

动态相关的集合
- 用户状态集合
- 播放状态集合
- 播放记录集合

```
用户状态集合
用于实现用户状态：
- 在线状态、在线但屏幕未显示状态、输入状态、离线状态
还附带用户当前的一些信息
- 用户当前正在播放的播放列表（关系字段）

播放状态集合
用于实现同步播放
用于获取正在播放的播放列表

播放记录集合
用于获取热门播放列表（即查询前500条，同播放列表同用户去重，计数多的即是热门的）
```


### 播放列表
- 播放列表页
- 播放列表选择页
- 播放列表详情页
- 播放项（基于文件的播放项）列表页、选择页
- 播放项（基于文件的播放项）详情页
- 播放项（基于链接的播放项）列表页、选择页
- 播放项（基于链接的播放项）详情页


```
播放列表页（播放列表列表页）
全部播放列表、我的播放列表
按更新时间排序

播放列表选择页
和播放列表页类似，区别是多了底栏确认栏等
主要用于在聊天中发送某个播放列表
用户在聊天中可发送播放列表，以达到推荐观看的目的

播放列表详情页
显示播放列表名、简介、关键词、封面
根据此播放列表是否正在播放，正在被几个人主持播放、显示几个卡片
有 跟随播放 按钮（在对应卡片中）
有 主持播放 按钮
有 独自播放 按钮
显示播放列表中的全部播放项
创建者可修改其详情、播放项
关于播放项修改，可在本页调整其排序、移除，添加操作即跳转至对应的播放项选择页



播放项（基于文件的播放项）列表页、选择页

播放项（基于文件的播放项）详情页

播放项（基于链接的播放项）列表页、选择页

播放项（基于链接的播放项）详情页
```
