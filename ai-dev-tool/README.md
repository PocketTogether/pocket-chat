# 📘 ai-dev-tool

这个目录专门放 **和 AI 协作相关的东西**。  
简单说：这里放的是“让 AI 更懂这个项目”的所有资料。

里面主要有两类内容：

1. **工具脚本**（现在只有一个）  
2. **提示词 / 规则 / 项目说明**（都放在 prompts/ 里）

---

# 📁 目录说明

```
ai-dev-tool/
│
├── dev-flatten-text-to-ai.js   # 把多个文件合并成一份给 AI 的输入
│
└── prompts/                    # 所有提示词、规则、项目说明都放这里
```

---

# 🧩 dev-flatten-text-to-ai.js 是干嘛的

一句话：

> **把多个文件合成一份长文本，方便复制给 AI。**

比如你想让 AI 同时理解：

- 项目结构  
- 你的编码习惯  
- i18n 规则  
- 分段式生成协议  
- 某个草稿  

你就可以这样用：

```powershell
node dev-flatten-text-to-ai.js `
E:/Project/pocket-chat/ai-dev-tool/prompts `
E:/Project/pocket-chat/note/草稿260113 `
E:/Project/pocket-chat/vue3/src/lib
```

脚本会把这些文件按顺序拼成一份文本，输出到终端，你复制给 AI 就行。

---

# 🧩 prompts/ 是干嘛的

这里放的都是：

- 你写给 AI 的提示词  
- 你的项目习惯  
- 你的代码风格  
- 你的生成协议  
- 你的草稿说明  

简单说：

> **这是 AI 的“学习资料库”。**

AI 生成代码前，你把 prompts/ 里的内容 flatten 一下，它就能按你的风格写代码。

---

# 🧠 为什么要有这个目录

因为你现在的开发方式是：

> **AI 参与整个项目，需要上下文、规则、协议。**

这些东西不能散落在各处，所以统一放在 ai-dev-tool 里，方便管理，也方便未来扩展。

---

# 🚀 怎么用

1. 写提示词、规则、说明 → 放到 prompts/  
2. 需要让 AI 理解项目 → 用 dev-flatten-text-to-ai.js 合并  
3. 把输出复制给 AI  
4. 让 AI 按你的协议生成代码  

---

# 📌 总结

`ai-dev-tool/` 的作用非常简单：

- **让 AI 更懂你的项目**
- **让 AI 写的代码更统一**
- **让你能控制 AI 的生成流程**
- **让提示词和规则有地方放**

未来你想扩展更多工具、更多协议，也都可以继续放在这里。
